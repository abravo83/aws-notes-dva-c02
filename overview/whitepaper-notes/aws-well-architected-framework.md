# AWS Well-Architected

A tool to learn, measure and build applications using architectural best practises.

## Overview

AWS Well-Architected is a tool to help cloud architects build secure, high-performing, resilient, and efficient infrastructure for different application types. It is build around six pillars –operational excellence, security reliability, performance efficiency, cost optimization, and sustainability–.

AWS Well-Architected Framework is formed by domain-specific lenses, hands-on labs, and the Well-Architected tool. It describes key concepts, design principles, and architectural best practises for designing and running workloads in the cloud. 

## AWS Well-Architected Six Pillars

### Operacional Excellence Pillar

Focuses on running and monitoring systems, and continually improving processes and procedures. The key topics are automating changes, responding to events, and defining standards to manage daily operations.

### Security Pillar

Focuses on protecting information and systems. Key topics include confidentiality and integrity of data, managing user permissions, and establishing controls to detect security events.

### Reliability Pillar

Focuses on workloads performing their intended functions and how to recover quickly from failure to meet demands. Key topics include distributed system design, recovery planning, and adapting to changing requirements.
